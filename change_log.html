1.2.1 (2016-11-09)

Micro release of RedDeer. Removed checkstyle

1.2.0 (2016-11-02)

RedDeer 1.2.0 supports Eclipse Neon (EE). This release contains mostly bug fixes and few enhancements. New and noteworthy features:

  • RedDeer log collector
  • Support for prioritized extensions
  • TreeViewerHandler improved to get more items with same text
  • Wait condition TextEditorContainsText
  • Changed visibility from internal to public of Server wait conditions

Around 37 issues in total were resolved in this version. For complete reference see resolved pull requests and resolved issues.

1.1.0 (2016-08-18)

RedDeer 1.1.0 supports Eclipse Neon (EE). This release contains mostly bug fixes and few enhancements. New and noteworthy features:

  • Extension to check whether an issue causing test failure has been fixed
  • Capability to kill a specific job
  • Implemented Import Maven Project wizard
  • Implemented HTML wizard
  • Managing archetype catalogs
  • New ShellIsAvailable wait condition
  • Many enhancements in TextEditor
  • Problems view enhancements

Around 70 issues in total were resolved in this version. For complete reference see resolved pull requests and resolved issues.

0.8.0 (2015-08-06)

RedDeer 0.8 supports Eclipse Mars (EE). This version is mainly about architecture improvements, related re-factoring and lot of bug fixes. In terms of these changes lots of underlying functionality went into o.j.r.common and mainly to o.j.r.core plugin. This helped us to keep architecture cleaner and to avoid cross dependency issues we have faced. Still you can also find multiple API and tooling features and enhancements in this release, for example:

You can also find some new features/API like:

  • Utility Project Wizard
  • Property Dialog
  • Log Parser
  • RedDeer Spy View
  • User Profile support

Now when you're creating test plugin you can simply add o.j.r.go dependency and start coding

Also many (~70) bug fixes and other issues (~180 in total) resolved here
